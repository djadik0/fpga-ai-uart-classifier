# AI Image Classifier on FPGA (SystemVerilog)

## Описание проекта

Этот проект представляет собой учебный аппаратный классификатор изображений на FPGA с реализацией на SystemVerilog Изображение размером 64×64 передаётся с ПК в FPGA по UART записывается во внутреннюю память, проходит через вычислительный RTL-конвейер и в конце устройство возвращает по UART один байт `class_id` — номер распознанной цифры.

Проект собран как единая RTL-система и включает:

- приём данных по UART;
- загрузку изображения во внутреннюю память;
- свёрточный слой;
- max-pooling;
- fully-connected слой;
- `argmax` для выбора итогового класса;
- передачу результата обратно на ПК;
- Python-часть для обучения сети и генерации `.mem` весов под RTL;
- Python-скрипт для связи между ПК и FPGA по UART.

Цель проекта — получить полностью рабочий учебный **RTL inference pipeline** без использования внешнего процессора в вычислительной части.


## Текущий статус проекта

На данный момент в проекте реализованы:

- верхний уровень `ai_top` для интеграции UART и вычислительного ядра;
- вычислительное ядро `top_module`;
- загрузка изображения 64×64 по UART;
- хранение весов свёрточного и fully-connected слоя в `.mem` файлах;
- полный проход данных: `UART RX -> image loader -> memory -> conv -> pooling -> FC -> argmax -> UART TX`;
- тестбенч полного уровня;
- Python-часть для обучения сети, экспорта весов и отправки изображений на FPGA;
- набор тестовых изображений в папке `mnist_png`;
- отдельная папка `network` с кодом сети и обучением;
- XDC-констрейнт под используемую плату.


## Общая архитектура

Поток данных в проекте:

```text
PC / Host
  │
  │ UART: 0xA5 + 4096 байт изображения
  ▼
[ uart_rx ]
  ▼
[ uart_image_loader ]
  ▼
[ memory_buffer ]
  ▼
[ conv_layer ] + [ conv_mac ]
  ▼
[ pooling_layer ]
  ▼
[ fc_layer ] + [ fc_weight_memory ]
  ▼
[ argmax_layer ]
  ▼
[ uart_tx ]
  │
  └──> class_id обратно на ПК
```


## Параметры текущей реализации

Основные параметры вычислительного ядра:

- входное изображение: 64×64
- ширина пикселя: 8 бит
- размер свёрточного ядра: 4×4
- число фильтров: 4
- ширина результата свёртки: 32 бита
- pooling: 2×2
- stride pooling: 2
- число выходных классов: 10

Расчёт размеров:

- после свёртки:

```text
64 - 4 + 1 = 61
```

- после pooling:

```text
(61 - 2) / 2 + 1 = 30
```

- итоговый размер признаков перед FC:

```text
4 × 30 × 30 = 3600
```

То есть fully-connected слой получает 3600 входных значений и формирует 10 выходных классов.



## Как работает проект

### 1. Приём данных по UART

FPGA ожидает стартовый байт:

```text
0xA5
```

После него с ПК передаются:

```text
64 × 64 = 4096 байт изображения
```

Формат UART:

- `CLK_FREQ = 50_000_000`
- `BAUD_RATE = 115200`
- формат кадра: **8N1**

### 2. Загрузка изображения

Модуль `uart_image_loader` принимает поток байтов и построчно записывает изображение в память.

Координаты обновляются в таком порядке:

- сначала `x`
- после конца строки увеличивается `y`

После записи последнего пикселя формируется сигнал окончания загрузки.

### 3. Свёрточный слой

`conv_layer` управляет свёрточным проходом:

- считывает веса нужного фильтра из `memory_buffer`;
- запрашивает фрагмент изображения `4×4`;
- передаёт фрагмент и веса в `conv_mac`;
- получает результат скалярного произведения;
- выдаёт поток свёрточных значений дальше в pooling.

`conv_mac` вычисляет:

```text
sum(A[i][j] * B[i][j])
```

где:

- `A` — фрагмент изображения;
- `B` — веса фильтра.

### 4. Pooling

`pooling_layer` выполняет **max-pooling 2×2** со stride 2 и сохраняет результат во внутреннюю память pooling-слоя.

### 5. Fully-connected слой

`fc_layer` последовательно читает значения после pooling и соответствующие веса из `fc_weight_memory`, после чего считает 10 выходных сумм — по одной на каждый класс.

### 6. Argmax

`argmax_layer` выбирает индекс максимального выходного значения и формирует итоговый `class_id`.

В текущей реализации:

- `class_id = 0` соответствует цифре `0`
- `class_id = 1` соответствует цифре `1`
- ...
- `class_id = 9` соответствует цифре `9`

### 7. Передача результата

После завершения вычислений `uart_tx` отправляет обратно на ПК **1 байт** — итоговый `class_id`.


## Верхний уровень проекта

### `ai_top.sv`

Это главный top-level проекта.

Он объединяет:

- UART-приёмник;
- UART-передатчик;
- модуль загрузки изображения;
- вычислительное ядро.

Внутри `ai_top.sv` реализован управляющий FSM, который проходит стадии:

- ожидание стартовой команды;
- запуск загрузки изображения;
- ожидание завершения загрузки;
- запуск AI-вычисления;
- ожидание завершения inference;
- запуск UART-передачи результата;
- ожидание окончания передачи.

### `top_module.sv`

Это основное вычислительное ядро.

Он связывает между собой:

- `memory_buffer`
- `conv_layer`
- `conv_mac`
- `pooling_layer`
- `fc_layer`
- `fc_weight_memory`
- `argmax_layer`


## Работа с весами нейросети

Для inference используются заранее подготовленные веса.

В проекте участвуют два основных `.mem` файла:

- `buffer_weight.mem` — веса свёрточных фильтров;
- `fc_weight.mem` — веса fully-connected слоя.

Они генерируются на стороне Python из обученной сети и затем используются RTL-модулями через `$readmemh(...)`.


## Обучение сети и подготовка весов

Папка `network` содержит Python-часть проекта.

Там находится:

- описание сети, соответствующей текущей RTL-архитектуре;
- код обучения;
- код экспорта весов в `.mem`;
- сохранённая обученная модель;
- служебная информация по квантованию.

Сеть обучается под текущую структуру inference pipeline на FPGA:

- вход `64×64`
- 1 свёрточный слой
- pooling
- fully-connected
- 10 классов

После обучения веса экспортируются в `.mem` формат для дальнейшего использования в RTL.


## Тестовые изображения

Папка `mnist_png` содержит изображения цифр, которые используются для проверки распознавания.

Это отдельная часть проекта для удобного тестирования Python-скриптами и для отправки изображений в FPGA по UART.


## Связь между ПК и FPGA

Для обмена с платой используется Python-скрипт из папки `code`.

Текущий сценарий работы:

1. ПК открывает COM-порт.
2. Отправляет стартовый байт `0xA5`.
3. Отправляет 4096 байт изображения.
4. Ждёт один байт ответа.
5. Считывает `class_id`.

Таким образом можно тестировать проект уже не только в симуляции, но и на реальной плате через внешний USB-UART адаптер.


## Аппаратная привязка

В проекте используется XDC-констрейнт для платы **Z7-Lite / XC7Z020**.

Основные сигналы верхнего уровня:

- `clk` — тактирование PL, **50 MHz**
- `rst` — reset
- `rx_i` — UART RX в PL
- `tx_o` — UART TX из PL
- `dbg_led` — отладочный светодиод

Используемые пины в текущем проекте:

- `clk` → `N18`
- `rst` → `P16`
- `rx_i` → `F17`
- `tx_o` → `F16`
- `dbg_led` → `P15`

## Запуск на FPGA

Типовой сценарий:

1. Сгенерировать или обновить веса в папке `network`.
2. Подставить актуальные `.mem` файлы в RTL-проект.
3. Собрать bitstream.
4. Загрузить bitstream в плату.
5. Подключить внешний USB-UART адаптер.
6. Запустить Python-скрипт отправки изображения.
7. Получить обратно `class_id`.


## Особенности текущей реализации

- проект полностью написан на **SystemVerilog**;
- вычислительная часть работает как последовательный аппаратный inference pipeline;
- reset в RTL — активный по низкому уровню;
- веса загружаются из `.mem` файлов;
- сеть и RTL согласованы по количеству классов и структуре слоёв;
- проект ориентирован именно на распознавание цифр;
- Python-часть используется только для подготовки весов и отправки изображений, а не для inference на устройстве.


## Ограничения текущей версии

Текущая версия проекта является учебной и в первую очередь демонстрирует полный RTL-путь обработки данных на FPGA: приём изображения, запись в память, аппаратный inference pipeline и передачу результата обратно на ПК.

Качество распознавания в текущей версии ограничено простой архитектурой сети и параметрами обучения. Проект лучше рассматривать как демонстрацию **RTL-интеграции, UART-связи, memory-based dataflow, работы с `.mem` весами и аппаратного inference pipeline**, а не как финальную высокоточную систему классификации.

Дальнейшее улучшение качества связано с:

- улучшением предобработки входных изображений;
- обучением строго под FPGA-формат входных данных;
- возможным усилением архитектуры сети;


## Что можно улучшать дальше

Направления развития:

## Итог

Этот проект представляет собой учебную систему распознавания изображений на FPGA, в которой:

- изображение передаётся с ПК по UART;
- сохраняется во внутреннюю память;
- проходит через RTL-конвейер из свёрточного, pooling и fully-connected слоёв;
- итоговый класс выбирается через `argmax`;
- результат возвращается обратно на ПК.

Проект охватывает сразу несколько частей одной системы:

- RTL-проектирование;
- интеграцию модулей на верхнем уровне;
- подготовку весов нейросети в Python;
- связь между ПК и FPGA;
- симуляцию и практическое тестирование на реальном устройстве.
