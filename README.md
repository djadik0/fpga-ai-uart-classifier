# AI Image Classifier on FPGA (SystemVerilog)

## Описание проекта

Этот проект представляет собой аппаратный **классификатор изображений на SystemVerilog**, в котором изображение передаётся в устройство по **UART**, далее обрабатывается конвейером из нескольких слоёв, после чего модуль возвращает **номер предсказанного класса** обратно по UART.

Проект построен как законченная RTL-система, включающая:
- приём изображения по UART;
- запись изображения во внутреннюю память;
- свёрточную обработку с использованием весов из `.mem` файлов;
- max-pooling;
- fully-connected слой;
- выбор класса через `argmax`;
- передачу результата обратно на ПК.

По структуре это **CNN-подобный аппаратный inference pipeline**, реализованный полностью на RTL без использования внешнего процессора.


## Основные возможности

- загрузка изображения размером **64×64** по UART;
- обработка изображения внутри FPGA/RTL-пайплайна;
- поддержка **10 выходных классов**;
- хранение весов в файлах инициализации памяти:
  - `buffer_weight.mem` — веса свёрточных фильтров;
  - `fc_weight.mem` — веса полносвязного слоя;
- возврат результата классификации одним байтом по UART;
- наличие отдельного тестбенча для проверки полного сценария работы от входного UART-пакета до ответа на выходе.


## Архитектура проекта

Общий поток данных в проекте:

```text
PC/Host
  │
  │ UART (команда + 4096 байт изображения)
  ▼
[ uart_rx ]
  ▼
[ uart_image_loader ]
  ▼
[ memory_buffer ]
  ▼
[ conv_layer ] + [ conv_mac ]
  ▼
[ pooling_layer ]
  ▼
[ fc_layer ] + [ fc_weight_memory ]
  ▼
[ argmax_layer ]
  ▼
[ uart_tx ]
  │
  └──> class_id обратно на PC/Host
```

### Верхний уровень

#### `ai_top.sv`
Главный top-level всего проекта.

Он объединяет:
- UART-приёмник `uart_rx`;
- UART-передатчик `uart_tx`;
- загрузчик изображения `uart_image_loader`;
- вычислительное ядро `top_module`.

Также в `ai_top.sv` реализован управляющий FSM со стадиями:
- ожидание стартовой команды;
- запуск загрузки изображения;
- ожидание окончания загрузки;
- запуск AI-обработки;
- ожидание завершения вычислений;
- запуск передачи результата;
- ожидание завершения UART TX.

Именно этот модуль удобно использовать как главный RTL top для симуляции и дальнейшей интеграции на плату.

#### `top_module.sv`
Основное вычислительное ядро классификатора.

Содержит и связывает между собой:
- `memory_buffer` — память изображения и весов свёртки;
- `conv_layer` — управление свёрточным проходом по изображению;
- `conv_mac` — вычисление скалярного произведения фрагмента и фильтра;
- `pooling_layer` — max-pooling;
- `fc_layer` — полносвязный слой;
- `fc_weight_memory` — память весов FC-слоя;
- `argmax_layer` — выбор индекса максимального значения.


## Структура вычислений

Параметры по умолчанию:

- размер входного изображения: **64×64**;
- размер ядра свёртки: **4×4**;
- количество фильтров: **4**;
- размер pooling-окна: **2×2**;
- stride pooling: **2**;
- число выходных классов: **10**.

### 1. Свёрточный слой

Модуль `conv_layer` последовательно:
- считывает веса текущего фильтра из `memory_buffer`;
- запрашивает из памяти очередной фрагмент изображения `4×4`;
- передаёт фрагмент и веса в `conv_mac`;
- сохраняет результат в карту признаков.

Так как вход имеет размер `64×64`, а окно свёртки `4×4`, размер выходной карты после свёртки:

```text
64 - 4 + 1 = 61
```

То есть каждый фильтр формирует карту признаков **61×61**, а всего карт **4**.

### 2. Pooling

Модуль `pooling_layer` выполняет **max-pooling 2×2** со stride 2.

Размер каждой карты после pooling:

```text
(61 - 2) / 2 + 1 = 30
```

На выходе получается массив признаков размера:

```text
4 × 30 × 30 = 3600 значений
```

### 3. Fully Connected слой

Модуль `fc_layer` последовательно разворачивает выход pooling-слоя и вычисляет значения для **10 выходных нейронов**.

Для каждого класса используются **3600 весов**, которые считываются из `fc_weight.mem` через модуль `fc_weight_memory`.

### 4. Argmax

Модуль `argmax_layer` выбирает индекс максимального выходного значения FC-слоя и формирует итоговый `class_id`.


## Формат обмена по UART

Протокол обмена, реализованный в `ai_top.sv`:

1. Хост отправляет **стартовый байт**:

```text
0xA5
```

2. После этого передаются **4096 байт изображения**:

```text
64 × 64 = 4096 пикселей
```

3. Модуль `uart_image_loader` записывает изображение во внутреннюю память построчно:
- сначала меняется `x`;
- после конца строки увеличивается `y`.

4. После загрузки автоматически запускается вычислительное ядро.

5. После завершения inference устройство передаёт **1 байт** — номер класса.

### Параметры UART по умолчанию

В модулях `uart_rx.sv` и `uart_tx.sv` используются параметры:
- `CLK_FREQ = 50_000_000`
- `BAUD_RATE = 115200`
- формат кадра: **8N1**

То есть интерфейс рассчитан на стандартный UART без parity:
- 1 старт-бит;
- 8 бит данных;
- 1 стоп-бит.


## Описание основных модулей

### `uart_rx.sv`
Аппаратный UART-приёмник.

Функции:
- синхронизация входного сигнала `rx_i`;
- детектирование старт-бита;
- приём 8 бит данных;
- проверка стоп-бита;
- выдача `rx_valid_o` и `rx_data_o`.

### `uart_tx.sv`
Аппаратный UART-передатчик.

Функции:
- запуск передачи по `start_tx_i`;
- передача старт-бита, 8 бит данных и стоп-бита;
- сигнализация занятости `busy_o`;
- формирование импульса `tx_done_o` после завершения передачи.

### `uart_image_loader.sv`
Модуль загрузки изображения из входного потока UART.

Функции:
- после команды старта переходит в режим загрузки;
- принимает каждый байт пикселя;
- формирует сигналы записи в память изображения;
- автоматически считает координаты `x/y`;
- выдаёт `image_done_o` после приёма последнего пикселя.

### `memory_buffer.sv`
Общая память проекта.

Содержит:
- память весов свёрточного слоя;
- память изображения;
- логику выдачи фрагмента `4×4` для свёртки.

Особенность модуля:
- свёрточные веса инициализируются из файла `buffer_weight.mem`;
- изображение загружается по UART во время работы;
- фрагмент для MAC формируется напрямую из массива `image_memory`.

### `conv_mac.sv`
MAC-ядро свёртки.

Выполняет накопление суммы произведений элементов:

```text
result = Σ (A[i][j] * B[i][j])
```

где:
- `A` — фрагмент изображения;
- `B` — веса фильтра.

### `conv_layer.sv`
Управляющий модуль свёрточного слоя.

Отвечает за:
- загрузку весов текущего фильтра;
- перебор всех позиций фрагмента по изображению;
- запуск `conv_mac`;
- сохранение результатов свёртки в выходной массив.

### `pooling_layer.sv`
Комбинаторный модуль max-pooling.

Для каждого окна `2×2` выбирает максимальное значение и записывает его в выходной массив.

### `fc_weight_memory.sv`
Память весов fully-connected слоя.

Весовые коэффициенты инициализируются из файла `fc_weight.mem`.

### `fc_layer.sv`
Полносвязный слой.

Функции:
- перебор всех 3600 входных признаков;
- последовательное умножение на вес текущего класса;
- накопление суммы;
- формирование 10 выходных значений `fc_out_o`.

### `argmax_layer.sv`
Модуль выбора итогового ответа.

Сравнивает значения всех выходных нейронов и возвращает индекс максимального значения — `class_id`.


## Тестирование

Для проекта подготовлен тестбенч полного уровня:

### `tb_ai_top.sv`
Тестбенч проверяет весь тракт работы системы:
- отправляет стартовую команду `0xA5`;
- передаёт изображение в DUT по UART;
- принимает ответ DUT через отдельный экземпляр `uart_rx`;
- сравнивает полученный класс с ожидаемым.

В тестбенче предусмотрены два сценария:

1. **Изображение из единиц** — все пиксели равны `1`;
2. **Шахматный шаблон 0/1**.

В обоих тестах ожидается ответ:

```text
class_id = 9
```

Это подтверждает корректную работу следующей цепочки:

```text
UART RX -> image loader -> memory -> conv -> pooling -> FC -> argmax -> UART TX
```

## Как запускать симуляцию

Ниже приведён типовой пример запуска в QuestaSim/ModelSim.

> Важно: файлы `buffer_weight.mem` и `fc_weight.mem` должны находиться в рабочей директории симуляции, чтобы команды `$readmemh(...)` смогли их открыть.

### Пример компиляции

```bash
vlog ai_top.sv \
     top_module.sv \
     uart_rx.sv \
     uart_tx.sv \
     uart_image_loader.sv \
     memory_buffer.sv \
     conv_layer.sv \
     conv_mac.sv \
     pooling_layer.sv \
     fc_layer.sv \
     fc_weight_memory.sv \
     argmax_layer.sv \
     tb_ai_top.sv
```

### Запуск

```bash
vsim tb_ai_top
run -all
```

### Ожидаемый результат

При успешной симуляции тестбенч должен вывести сообщения вида:

```text
PASS: ai_top processed both UART images correctly
```

## Особенности реализации

- проект написан на **SystemVerilog**;
- основной сценарий работы ориентирован на **inference**, а не обучение;
- веса свёртки и FC-слоя заранее подготовлены и подгружаются через `.mem` файлы;
- загрузка изображения реализована как внешний поток байтов через UART;
- вычисления выполняются последовательно и аппаратно управляются FSM-модулями;
- reset в проекте **активный по низкому уровню** (`rst`, проверка `if (!rst)`).

## Итог

Данный проект — это аппаратная система классификации изображений на FPGA/RTL, в которой входные данные приходят по UART, проходят через свёрточно-полносвязный конвейер и возвращаются как готовый номер класса.

Он демонстрирует не только отдельные модули, но и **полную интеграцию интерфейса передачи данных, памяти, вычислительных блоков и верификации**.
